# ChatApp

A real-time chat application built with **.NET 8**: a WPF desktop client talking to an
ASP.NET Core server over **SignalR** (chosen over raw WebSockets - built into ASP.NET Core,
reliable and well known to me) and **REST** (users & history),
with **MySQL** persistence via **Entity Framework Core**.

```
┌────────────────┐       SignalR  +  REST         ┌────────────────┐   EF Core   ┌───────┐
│ ChatApp.Client │ ─────────────────────────────► │ ChatApp.Server │ ──────────► │ MySQL │
│  (WPF, MVVM)   │ ◄───────────────────────────── │ (ASP.NET Core) │             └───────┘
└────────────────┘      broadcast to all clients  └────────────────┘
        ▲                                                 ▲
        └──────────── ChatApp.Core (shared DTOs) ─────────┘
                                                            (schematic generated by Claude)
```

- Join with a username (new users are created, returning users are recognized)
- Messages are persisted and pushed instantly to every connected client

## Prerequisites

- [.NET 8 SDK](https://dotnet.microsoft.com/download/dotnet/8.0)
- Windows (the client is WPF)
- MySQL 8 - either via [Docker](https://www.docker.com/) (easiest, step 1 below)
  or a local MySQL installation

## Getting started

**1. Start MySQL.** With Docker (from the repository root):

```
docker compose up -d
```

This starts MySQL 8 on port 3306 with password `chatapp-dev` and creates the `chatapp`
database. 

*Alternatively*, use an existing local MySQL 8 install - just know its root
password (the database itself is created automatically on first run).

> Using a local MySQL that already runs on port 3306? Skip Docker - the two would
> collide on the port. Either works; the app just needs *a* MySQL on 3306.

**2. Configure the connection string** (the repo contains no real password).
From the repository root:

```
dotnet user-secrets set "ConnectionStrings:ChatDb" "server=localhost;port=3306;user=root;password=chatapp-dev;database=chatapp" --project ChatApp.Server
```

Replace `chatapp-dev` with your own root password if you're not using Docker.
(Environment variable `ConnectionStrings__ChatDb` works too.)

**3. Run the server** - it applies database migrations automatically on startup:

```
dotnet run --project ChatApp.Server --launch-profile http
```

The API + Swagger UI are now at http://localhost:5095/swagger.

**4. Run one or more clients** (each in its own terminal):

```
dotnet run --project ChatApp.Client
```

Open two clients, join with two different usernames, and chat - messages appear in both
windows in real time.

## Project structure

| Project | Description |
|---|---|
| `ChatApp.Core` | Shared DTOs (the wire contract between client and server), implemented only for design purposes, I know this is a small scale and was not necessary, but it's still the proper way |
| `ChatApp.Server` | ASP.NET Core: REST controllers, SignalR hub, EF Core + migrations |
| `ChatApp.Client` | WPF desktop client (MVVM): SignalR + HTTP services, ViewModels, XAML |

## Not implemented

- Authentication (small scale, out of scope), usernames are claimed on a first-come basis
- Chat rooms - just a single global chat room, single-server deployment (no backplane)
- No automated tests yet
